## 🧠 Summary

> In this lab, I explored Wazuh, an all-in-one, open-source security platform designed for threat detection, security monitoring, log analysis, and incident response. I gained
hands-on experience navigating the Wazuh environment and learning how its components work together to provide centralized visibility across endpoints and infrastructure.

Throughout the lab, I worked with key Wazuh capabilities such as log collection, security event monitoring, agent management, and alert analysis. I learned how Wazuh ingests and 
processes security data from monitored systems, enabling analysts to identify suspicious activity and investigate potential threats more effectively.

I also explored how Wazuh integrates with security monitoring workflows by using rules, dashboards, and detection mechanisms to generate actionable alerts. This provided valuable 
insight into how organizations leverage centralized monitoring platforms to improve their security posture and detect malicious activity across their environments.

This lab reinforced the importance of security information and event management (SIEM) and endpoint monitoring solutions, improving my understanding of how Wazuh supports threat 
detection, incident investigation, compliance monitoring, and overall security operations in real-world SOC environments.

*** Auditing Commands on Linux with Wazuh ***

> In this section, I used the auditd application to monitor command execution within the Wazuh instance. I first installed auditd on the VM and then manually added an audit rule.
One of the command syntaxes used was:

-a exit,always -F arch=64 -F euid=0 -S execve -k audit-wazuh-c

This rule logs every command executed by a process running as root on a 64-bit Linux system.

I then informed auditd of the new rule by running the command:

sudo auditctl -R /etc/audit/rules.d/audit.rules

Next, I ran the command:

sudo nano /var/ossec/etc/ossec.conf

This allowed the Wazuh agent to detect the new log file generated by auditd.

I then configured the ossec.conf file with the following entry:

<localfile>
<location>/var/log/audit/audit.log</location>
<log_format>audit</log_format>
</localfile>

This configuration sends the audit log file to the Wazuh management server.
